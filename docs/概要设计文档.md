# 大学生资源信息分享平台 - 概要设计文档

## 1. 系统架构设计

### 1.1 总体架构
本系统采用前后端分离的分布式架构，整体架构如下：

```
┌─────────────────────────────────────────────────────────┐
│                      客户端层                             │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │  Web 端   │  │ 小程序端 │  │ Harmony  │              │
│  │ Vue3    │  │  Uniapp  │  │   APP    │              │
│  └──────────┘  └──────────┘  └──────────┘              │
└─────────────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────┐
│                      网关层                               │
│              Nginx / Spring Cloud Gateway               │
│         (负载均衡、路由转发、限流、鉴权)                    │
└─────────────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────┐
│                   应用服务层                              │
│  ┌──────────────────────────────────────────────────┐  │
│  │          Spring Boot 3.x 应用集群                  │  │
│  │  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐   │  │
│  │  │用户服务│ │优惠服务│ │社区服务│ │消息服务│   │  │
│  │  └────────┘ └────────┘ └────────┘ └────────┘   │  │
│  └──────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────┐
│                   数据层                                  │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │  MySQL   │  │  Redis   │  │  文件存储  │              │
│  │  主从复制 │  │  缓存集群 │  │  OSS/MinIO│             │
│  └──────────┘  └──────────┘  └──────────┘              │
└─────────────────────────────────────────────────────────┘
```

### 1.2 技术架构

#### 1.2.1 后端架构
- **核心框架**: Spring Boot 3.x
- **ORM 框架**: MyBatis-Plus 3.5.x
- **数据库**: MySQL 8.0（主从复制）
- **缓存**: Redis 7.x
- **认证**: JWT + Spring Security
- **API 文档**: Knife4j (Swagger 3)
- **日志**: Logback + SLF4J
- **构建工具**: Maven 3.8+

#### 1.2.2 前端架构
- **Web 管理端**: Vue 3 + Element Plus + TypeScript
- **小程序端**: Uniapp + Vue 3
- **Harmony 端**: HarmonyOS SDK + ArkTS

### 1.3 系统分层

```
Controller 层  →  接收请求，参数校验，返回响应
     ↓
Service 层    →  业务逻辑处理，事务控制
     ↓
Mapper 层     →  数据访问，SQL 执行
     ↓
Entity 层     →  数据实体，ORM 映射
```

### 2.2 核心模块说明

#### 2.2.1 用户模块 (user)
- **职责**: 用户注册、登录、信息管理、认证授权
- **核心类**:
  - UserController: 用户接口控制器
  - UserService: 用户业务逻辑
  - UserMapper: 用户数据访问
  - User: 用户实体
  - JwtTokenProvider: JWT 令牌生成与验证

#### 2.2.2 优惠模块 (deal)
- **职责**: 优惠信息发布、浏览、管理
- **核心类**:
  - DealController: 优惠接口控制器
  - DealService: 优惠业务逻辑
  - DealMapper: 优惠数据访问
  - Deal: 优惠信息实体
  - DealCategory: 优惠分类实体

#### 2.2.3 社区模块 (community)
- **职责**: 内容发布、互动、评论
- **核心类**:
  - PostController: 帖子接口控制器
  - PostService: 帖子业务逻辑
  - CommentService: 评论业务逻辑
  - Post: 帖子实体
  - Comment: 评论实体

#### 2.2.4 消息模块 (message)
- **职责**: 消息通知、私信
- **核心类**:
  - MessageController: 消息接口控制器
  - MessageService: 消息业务逻辑
  - Notification: 通知实体

#### 2.2.5 搜索模块 (search)
- **职责**: 全局搜索、搜索建议
- **核心类**:
  - SearchController: 搜索接口控制器
  - SearchService: 搜索业务逻辑

#### 2.2.6 管理模块 (admin)
- **职责**: 后台管理、内容审核、数据统计
- **核心类**:
  - AdminController: 管理接口控制器
  - AdminService: 管理业务逻辑

## 4. 接口设计

### 4.1 接口规范

#### 4.1.1 请求格式
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

#### 4.1.2 错误码规范
- 200: 成功
- 400: 请求参数错误
- 401: 未授权
- 403: 禁止访问
- 404: 资源不存在
- 500: 服务器内部错误

### 4.2 核心接口

#### 4.2.1 用户接口
- POST /api/user/register - 用户注册
- POST /api/user/login - 用户登录
- GET /api/user/info - 获取用户信息
- PUT /api/user/info - 更新用户信息
- POST /api/user/logout - 用户登出

#### 4.2.2 优惠接口
- GET /api/deal/list - 获取优惠列表
- GET /api/deal/{id} - 获取优惠详情
- POST /api/deal - 发布优惠
- PUT /api/deal/{id} - 更新优惠
- DELETE /api/deal/{id} - 删除优惠
- POST /api/deal/{id}/favorite - 收藏优惠

#### 4.2.3 社区接口
- GET /api/post/list - 获取帖子列表
- GET /api/post/{id} - 获取帖子详情
- POST /api/post - 发布帖子
- POST /api/post/{id}/like - 点赞
- POST /api/post/{id}/comment - 评论

详见《API 接口文档》

## 5. 安全设计

### 5.1 认证机制
- 采用 JWT Token 进行身份认证
- Token 有效期 2 小时，Refresh Token 有效期 7 天
- 敏感操作需二次验证

### 5.2 权限控制
- 基于 RBAC 的权限模型
- 接口级别权限控制
- 数据级别权限控制

### 5.3 数据安全
- 密码 BCrypt 加密存储
- 敏感信息脱敏展示
- SQL 注入防护
- XSS 攻击防护

### 5.4 接口安全
- 接口限流（Rate Limiting）
- 防重放攻击
- 请求签名验证
