# 大学生资源信息分享平台 - 数据库设计文档

## 1. 数据库概述

### 1.1 数据库信息
- **数据库名称**: student_deal_share
- **数据库版本**: MySQL 8.0+
- **字符集**: utf8mb4
- **排序规则**: utf8mb4_general_ci
- **存储引擎**: InnoDB

### 1.2 设计规范
- 所有表使用 InnoDB 引擎
- 主键统一使用 id，自增 BIGINT
- 创建时间 (created_at) 和更新时间 (updated_at) 作为标准字段
- 逻辑删除使用 deleted 字段（0-未删除，1-已删除）
- 索引命名规范：idx_字段名，uk_字段名（唯一索引）

### 2.1 用户相关表

#### 2.1.1 用户表 (t_user)

CREATE TABLE `t_user` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '用户 ID',
  `username` VARCHAR(50) NOT NULL COMMENT '用户名',
  `password` VARCHAR(100) NOT NULL COMMENT '密码 (加密)',
  `nickname` VARCHAR(50) DEFAULT NULL COMMENT '昵称',
  `avatar` VARCHAR(255) DEFAULT NULL COMMENT '头像 URL',
  `gender` TINYINT DEFAULT 0 COMMENT '性别 0-未知 1-男 2-女',
  `phone` VARCHAR(20) DEFAULT NULL COMMENT '手机号',

  `school` VARCHAR(100) DEFAULT NULL COMMENT '学校',
  `college` VARCHAR(100) DEFAULT NULL COMMENT '学院',
  `student_id` VARCHAR(50) DEFAULT NULL COMMENT '学号',

  `last_login_time` DATETIME DEFAULT NULL COMMENT '最后登录时间',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted` TINYINT DEFAULT 0 COMMENT '删除标识 0-未删除 1-已删除',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`),
  UNIQUE KEY `uk_phone` (`phone`),
  UNIQUE KEY `uk_email` (`email`),
  KEY `idx_nickname` (`nickname`),

## 4. 索引设计

### 4.1 索引策略
- 主键索引：所有表的主键
- 唯一索引：用户名、手机号、邮箱等唯一字段
- 普通索引：查询条件频繁的字段
- 复合索引：多条件联合查询的字段

### 4.2 核心索引列表

| 表名 | 索引名 | 字段 | 类型 | 说明 |
|------|--------|------|------|------|
| t_user | uk_username | username | 唯一 | 用户名唯一 |
| t_user | uk_phone | phone | 唯一 | 手机号唯一 |
| t_user | idx_school | school | 普通 | 学校查询 |
| t_deal | idx_user_id | user_id | 普通 | 用户发布 |
| t_deal | idx_category_id | category_id | 普通 | 分类查询 |
| t_deal | idx_status | status | 普通 | 状态筛选 |
