# 大学生资源信息分享平台 - 需求分析文档

## 1. 项目概述

### 1.1 项目背景
当前电商平台、品牌方的营销活动形式多样（预售抢购、优惠券、限时折扣等），但普通用户获取优质活动信息的渠道分散，且抢购环节易因操作慢、信息差错失福利。同时，很多用户有分享自己"薅羊毛"经验、抢购成功案例的需求，却缺乏专属的垂直分享平台。

### 1.2 项目目标
构建一个面向大学生的资源信息分享平台，实现：
- 聚合各类优惠活动信息
- 提供快速抢购通道
- 搭建用户经验分享社区
- 打造垂直社交生态圈

### 1.3 目标用户
- 主要用户群体：在校大学生
- 次要用户群体：毕业生、教职工

## 2. 功能需求

### 2.1 用户模块

#### 2.1.1 用户注册与登录
- 支持手机号注册/登录
- 支持微信授权登录（小程序端）
- 支持账号密码登录
- 支持验证码登录
- 忘记密码功能

#### 2.1.2 用户信息管理
- 个人信息编辑（昵称、头像、性别、学校等）
- 实名认证（可选）
- 我的关注/粉丝
- 我的消息通知

#### 2.1.3 用户等级体系
- 积分系统
- 等级标识
- 等级权益

### 2.2 优惠信息模块

#### 2.2.1 优惠信息发布
- 发布优惠活动信息
- 支持多种优惠类型（预售、折扣、优惠券等）
- 上传活动图片/视频
- 设置活动时间
- 添加活动标签

#### 2.2.2 优惠信息浏览
- 首页推荐流
- 分类浏览
- 搜索功能（关键词、标签）
- 筛选功能（时间、热度、类型）
- 排序功能（最新、最热、最多优惠）

#### 2.2.3 优惠信息详情
- 活动详细信息展示
- 参与方式说明
- 直达链接/二维码
- 倒计时提醒
- 分享功能

#### 2.2.4 优惠信息管理
- 我的发布管理（编辑、删除、上下架）
- 收藏管理
- 浏览历史

### 2.3 社区分享模块

#### 2.3.1 内容发布
- 图文分享（薅羊毛经验、抢购案例）
- 支持富文本编辑
- 添加话题标签
- @其他用户
- 关联优惠信息

#### 2.3.2 内容互动
- 点赞
- 评论（支持回复）
- 转发/分享
- 收藏

#### 2.3.3 内容浏览
- 关注动态
- 热门内容推荐
- 话题广场
- 搜索功能

### 2.4 消息通知模块

#### 2.4.1 通知类型
- 系统通知
- 互动通知（点赞、评论、关注）
- 活动提醒（即将开始、库存预警）
- 私信消息

#### 2.4.2 通知管理
- 未读/已读标记
- 批量删除
- 通知设置

### 2.5 搜索模块

#### 2.5.1 搜索功能
- 全局搜索
- 搜索建议
- 搜索历史
- 热门搜索

#### 2.5.2 搜索结果
- 综合排序
- 分类筛选
- 结果高亮

### 2.6 后台管理模块

#### 2.6.1 用户管理
- 用户列表
- 用户详情
- 用户状态管理（禁用/启用）
- 用户等级调整

#### 2.6.2 内容管理
- 内容审核
- 内容推荐
- 违规内容处理
- 敏感词管理

#### 2.6.3 运营管理
- 活动管理
- 数据统计
- 系统配置

## 3. 非功能需求

### 3.1 性能需求
- 页面响应时间 < 2 秒
- 接口响应时间 < 500ms
- 支持并发用户数 ≥ 1000
- 数据库查询优化，关键查询 < 100ms

### 3.2 安全需求
- 用户密码加密存储（BCrypt）
- JWT Token 认证
- 接口防刷限制
- SQL 注入防护
- XSS 攻击防护
- 敏感信息脱敏

### 3.3 可用性需求
- 系统可用性 ≥ 99%
- 支持负载均衡
- 数据库主从复制
- 定期数据备份

### 3.4 兼容性需求
- 后端：Java 17+
- 前端：支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 移动端：HarmonyOS、微信小程序
- 响应式设计，支持多端适配

## 4. 技术栈

### 4.1 后端技术栈

- Maven（构建工具）

### 4.2 前端技术栈

- Element Plus（Web 管理端）

- Postman/Apifox（接口测试）
- Selenium（Web UI 自动化）
- Appium（移动端自动化）
