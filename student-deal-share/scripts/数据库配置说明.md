# 数据库配置说明

## 数据库驱动选择

本项目默认使用 **MariaDB** 驱动，同时预留 **MySQL** 驱动支持。

### 为什么选择 MariaDB？

1. **更好的兼容性**: MariaDB 是 MySQL 的分支，完全兼容 MySQL
2. **更好的性能**: 在某些场景下性能优于 MySQL
3. **开源友好**: 完全开源，无商业许可问题
4. **持续更新**: 社区活跃，更新频繁

### 驱动版本

- **MariaDB**: `org.mariadb.jdbc:mariadb-java-client:3.3.2`
- **MySQL**: `com.mysql:mysql-connector-j:8.0.33` (备用)

## 连接池配置

使用 **阿里巴巴 Druid** 连接池

### Druid 优势

1. **强大的监控功能**: 内置 StatViewServlet，提供详细的监控页面
2. **优秀的性能**: 国内广泛使用，性能卓越
3. **完善的配置**: 支持多种配置选项
4. **SQL 监控**: 支持 SQL 执行监控和慢查询日志

### Druid 版本

- `com.alibaba:druid-spring-boot-3-starter:1.2.20`

## 配置说明

### 开发环境配置 (application-dev.yml)

```yaml
spring:
  datasource:
    # MariaDB 驱动 (默认使用)
    url: jdbc:mariadb://localhost:3306/student_deal_share?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: root
    driver-class-name: org.mariadb.jdbc.Driver
    type: com.alibaba.druid.pool.DruidDataSource
    
    # MySQL 驱动 (备用)
    # url: jdbc:mysql://localhost:3306/student_deal_share?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    # username: root
    # password: root
    # driver-class-name: com.mysql.cj.jdbc.Driver
    # type: com.alibaba.druid.pool.DruidDataSource
  
  druid:
    initial-size: 5              # 初始连接数
    min-idle: 5                  # 最小空闲连接数
    max-active: 20               # 最大活跃连接数
    max-wait: 60000              # 获取连接最大等待时间 (ms)
    time-between-eviction-runs-millis: 60000  # 检测间隔
    min-evictable-idle-time-millis: 300000    # 最小空闲时间
    validation-query: SELECT 1   # 验证查询
    test-while-idle: true        # 空闲时检测
    test-on-borrow: false        # 借出时不检测
    test-on-return: false        # 归还时不检测
    pool-prepared-statements: true          # 开启 PSCache
    max-pool-prepared-statement-per-connection-size: 20  # 每个连接的最大 PSCache 大小
    
    # Druid 监控配置
    stat-view-servlet:
      enabled: true              # 启用监控页面
      url-pattern: /druid/*      # 监控页面路径
      reset-enable: false        # 禁止重置
      login-username: admin      # 监控页面登录用户名
      login-password: admin      # 监控页面登录密码
    
    # Web 监控配置
    web-stat-filter:
      enabled: true
      url-pattern: /*
      exclusions: "*.js,*.gif,*.jpg,*.png,*.css,*.ico,/druid/*"
```

## 切换数据库驱动

### 从 MariaDB 切换到 MySQL

1. 编辑配置文件（`application-dev.yml` 或 `application-prod.yml`）

2. 注释掉 MariaDB 配置：
```yaml
# url: jdbc:mariadb://localhost:3306/student_deal_share?...
# username: root
# password: root
# driver-class-name: org.mariadb.jdbc.Driver
```

3. 取消 MySQL 配置的注释：
```yaml
url: jdbc:mysql://localhost:3306/student_deal_share?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
username: root
password: root
driver-class-name: com.mysql.cj.jdbc.Driver
```

4. 如果使用 MySQL，需要在父工程 pom.xml 中取消 MySQL 驱动的注释：
```xml
<!-- 取消这个注释 -->
<dependency>
    <groupId>com.mysql</groupId>
    <artifactId>mysql-connector-j</artifactId>
    <version>${mysql.version}</version>
</dependency>

<!-- 注释掉这个 -->
<!--
<dependency>
    <groupId>org.mariadb.jdbc</groupId>
    <artifactId>mariadb-java-client</artifactId>
    <version>${mariadb.version}</version>
</dependency>
-->
```

## 数据库初始化

### 使用 SQL 脚本初始化

```bash
# 方法 1: 使用命令行
mariadb -u root -p student_deal_share < scripts/init.sql

# 或者使用 MySQL
mysql -u root -p student_deal_share < scripts/init.sql
```

### 使用 Docker 初始化

如果使用 Docker 部署，可以将 SQL 脚本放在容器的初始化目录：

```yaml
volumes:
  - ./scripts/init.sql:/docker-entrypoint-initdb.d/init.sql
```

## 连接池调优建议

### 开发环境
- 连接数可以设置较小（5-10）
- 开启详细的日志和监控
- 开启 SQL 输出

### 生产环境
- 根据并发量调整连接数（20-100）
- 关闭详细的日志输出
- 开启慢查询监控
- 设置合理的超时时间

## 常见问题

### 1. 连接超时
```
Cannot get connection for this connection
```
**解决方案**: 增加 `max-active` 和 `max-wait` 的值

### 2. 连接泄露
```
connection is not closed
```
**解决方案**: 开启 Druid 的连接泄露检测
```yaml
remove-abandoned: true
remove-abandoned-timeout: 1800
log-abandoned: true
```

### 3. 数据库版本不兼容
**解决方案**: 确保驱动版本与数据库版本匹配

### 4. 时区问题
```
The server time zone value is unrecognized
```
**解决方案**: 在连接 URL 中添加 `serverTimezone=Asia/Shanghai`

## 性能优化

### 1. 开启 PSCache
```yaml
pool-prepared-statements: true
max-pool-prepared-statement-per-connection-size: 50
```

### 2. 合理设置连接数
根据服务器配置和并发量设置：
- CPU 核数 * 2 + 1
- 或者根据压测结果调整

### 3. 开启监控
生产环境建议开启监控，但关闭详细日志

### 4. 定期清理
设置合理的空闲连接回收时间
