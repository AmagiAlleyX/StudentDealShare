# 大学生资源信息分享平台（经济）需求分析文档

## 1. 项目背景

当前电商平台、品牌方的营销活动形式多样（预售抢购、优惠券薅羊毛、限时折扣等），但普通用户获取优质活动信息的渠道分散，且抢购环节易因操作慢、信息差错失福利；同时，很多用户有分享自己"薅羊毛"经验、抢购成功案例的需求，却缺乏专属的垂直分享平台。

基于上述问题，构建一个大学生资源信息分享平台，旨在聚合校园可触达的优惠信息，并提供经验分享与抢购互助能力。

## 2. 功能需求

### 2.1 优惠信息聚合

| 功能点 | 描述 |
|-------|------|
| 活动分类 | 支持数码、生活、外卖、出行、课程等分类 |
| 来源标注 | 标注活动来源（京东、淘宝、美团、饿了么、银行APP等） |
| 时效性提醒 | 提供活动倒计时、即将开始等提醒功能 |
| 活动详情 | 展示活动规则、参与方式、有效期等详细信息 |
| 活动搜索 | 支持关键词搜索和筛选功能 |

### 2.2 薅羊毛经验分享

| 功能点 | 描述 |
|-------|------|
| 用户发帖 | 支持图文、链接等形式的经验分享 |
| 抢购成功案例 | 分享抢购成功的案例和技巧 |
| 评论互动 | 支持用户对帖子进行评论和互动 |
| 避坑提醒 | 提供活动中的注意事项和避坑指南 |
| 帖子点赞 | 支持用户对有价值的帖子进行点赞 |

### 2.3 抢购协作

| 功能点 | 描述 |
|-------|------|
| 活动收藏 | 支持用户收藏感兴趣的活动 |
| 活动提醒 | 对收藏的活动设置提醒，避免错过 |
| 抢购前预检清单 | 提供抢购前的准备事项清单 |
| 互助抢购请求 | （可选）支持用户发起互助抢购请求 |

### 2.4 校园特色

| 功能点 | 描述 |
|-------|------|
| 校内周边优惠 | 提供校内周边的优惠信息（打印、食堂、超市等） |
| 学生认证专属活动 | 推荐学生认证后可参与的专属活动 |
| 按学校/地区筛选 | 支持按学校或地区筛选优惠信息 |

### 2.5 用户管理

| 功能点 | 描述 |
|-------|------|
| 用户注册/登录 | 支持手机号、微信等方式注册和登录 |
| 学生认证 | 提供学生身份认证功能 |
| 个人中心 | 展示用户发布的帖子、收藏的活动等 |
| 消息通知 | 接收活动提醒、评论回复等消息 |

## 3. 非功能需求

### 3.1 性能需求

| 需求点 | 描述 |
|-------|------|
| 响应时间 | 页面加载时间不超过2秒，接口响应时间不超过500ms |
| 并发处理 | 支持至少1000用户同时在线访问 |
| 数据处理 | 能够处理和展示大量的优惠信息和用户帖子 |

### 3.2 可靠性需求

| 需求点 | 描述 |
|-------|------|
| 系统可用性 | 系统可用性达到99.9% |
| 数据备份 | 定期对数据进行备份，确保数据安全 |
| 错误处理 | 系统能够优雅处理各种异常情况，提供友好的错误提示 |

### 3.3 安全性需求

| 需求点 | 描述 |
|-------|------|
| 用户认证 | 采用安全的认证机制，保护用户账号安全 |
| 数据加密 | 对敏感数据进行加密存储 |
| 权限控制 | 确保用户只能访问和操作自己有权限的资源 |
| 防止恶意攻击 | 采取措施防止SQL注入、XSS等常见攻击 |

### 3.4 易用性需求

| 需求点 | 描述 |
|-------|------|
| 界面设计 | 界面简洁美观，操作直观易懂 |
| 响应式布局 | 支持手机、平板等多端适配 |
| 导航清晰 | 提供清晰的导航结构，方便用户快速找到所需功能 |
| 操作反馈 | 提供及时的操作反馈，增强用户体验 |

### 3.5 可维护性需求

| 需求点 | 描述 |
|-------|------|
| 代码规范 | 遵循统一的代码规范，提高代码可读性和可维护性 |
| 模块化设计 | 采用模块化设计，便于功能扩展和维护 |
| 文档完善 | 提供详细的开发文档和用户文档 |
| 日志记录 | 记录系统运行日志，便于问题排查 |

## 4. 数据需求

### 4.1 数据实体

| 实体名称 | 描述 | 主要字段 |
|---------|------|----------|
| 用户 | 平台用户 | id, 用户名, 手机号, 微信openid, 学生认证状态, 创建时间 |
| 活动 | 优惠活动 | id, 标题, 描述, 分类, 来源, 开始时间, 结束时间, 参与方式, 链接, 发布时间 |
| 帖子 | 用户分享的经验 | id, 标题, 内容, 图片, 链接, 发布用户id, 发布时间, 点赞数, 评论数 |
| 评论 | 对帖子的评论 | id, 帖子id, 用户id, 内容, 评论时间 |
| 收藏 | 用户收藏的活动 | id, 用户id, 活动id, 收藏时间 |
| 点赞 | 用户对帖子的点赞 | id, 用户id, 帖子id, 点赞时间 |
| 提醒 | 用户设置的活动提醒 | id, 用户id, 活动id, 提醒时间, 状态 |

### 4.2 数据存储

- 数据库：MySQL
- 缓存：Redis（可选，用于提高热门活动和帖子的访问速度）
- 文件存储：OSS（可选，用于存储用户上传的图片）

## 5. 技术栈需求

| 模块 | 技术 |
|------|------|
| 后端 | Java Spring Boot框架，使用MyBatis操作数据库 |
| 前端 | Uniapp（以微信小程序为准） |
| 移动端 | Harmony（手机/平板多端适配） |
| 数据库 | MySQL |
| 测试 | 单元测试、接口测试（PostMan/Apifox）、前端UI自动化（Selenium/Appium） |

## 6. 范围限定

- 本项目主要面向大学生群体，专注于优惠信息分享和抢购互助
- 不涉及电商交易功能，仅提供信息分享和链接跳转
- 初期版本以微信小程序为主要前端载体，后续可扩展到其他平台

## 7. 验收标准

### 7.1 功能验收

- 所有核心功能均已实现，且运行正常
- 用户能够正常注册、登录和进行学生认证
- 优惠信息能够及时更新和展示
- 用户能够正常发布、查看和互动帖子
- 抢购提醒功能能够正常工作

### 7.2 性能验收

- 页面加载时间不超过2秒
- 接口响应时间不超过500ms
- 系统能够支持1000用户同时在线访问

### 7.3 安全验收

- 用户认证和授权机制安全可靠
- 敏感数据得到有效保护
- 系统能够抵御常见的安全攻击

### 7.4 易用性验收

- 界面设计美观，操作直观
- 支持多端适配
- 用户能够快速找到所需功能

## 8. 风险评估

| 风险点 | 描述 | 应对措施 |
|-------|------|----------|
| 信息源可靠性 | 优惠信息来源可能存在虚假或过期信息 | 建立信息审核机制，对用户提交的信息进行审核 |
| 系统性能 | 随着用户量增加，系统性能可能下降 | 采用缓存机制，优化数据库查询，考虑分布式部署 |
| 安全风险 | 用户数据和活动信息可能面临安全威胁 | 加强系统安全措施，定期进行安全审计 |
| 运营风险 | 平台内容质量可能参差不齐 | 建立内容管理机制，鼓励高质量内容的创作和分享 |
